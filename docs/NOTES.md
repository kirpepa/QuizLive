# Пояснительная записка

**Проект:** VK Квиз — веб-приложение для проведения квизов в реальном времени (MVP).

## 1. Задача

Разработать MVP веб-приложения для проведения квизов: организатор создаёт и
настраивает квиз, запускает комнату; участники подключаются по коду и отвечают
синхронно в реальном времени; по завершении все видят лидерборд, результаты
сохраняются.

## 2. Обоснование выбора средств разработки

| Средство | Почему выбрано |
|----------|----------------|
| **React + Vite** | Компонентная модель хорошо ложится на набор экранов (кабинет, редактор, ведущий, игрок). Vite даёт мгновенный dev-сервер и простую сборку — быстрее, чем полноценный Next.js, а SSR для этого приложения не требуется. |
| **Tailwind CSS** | Быстрая, единообразная вёрстка. Позволила без лишнего кода оформить интерфейс в фирменном стиле VK (синяя палитра, системная типографика, аккуратные карточки). |
| **Node.js + Express** | Единый язык (JavaScript) на клиенте и сервере, низкий порог входа, большая экосистема. Express достаточно лёгок для REST-слоя MVP. |
| **Socket.IO** | Готовые «комнаты», авто-переподключение, ack-колбэки и fallback на polling. Идеально ложится на трансляцию вопросов и live-лидерборд; закрывает требование про WebSocket. |
| **SQLite + Prisma** | Нулевая настройка окружения (файл вместо сервера БД) — приложение запускается на любой машине без Docker. Prisma даёт типобезопасные запросы и миграции, а схема совместима с PostgreSQL: переход в продакшен — смена одной строки datasource. |
| **JWT (access/refresh)** | Простое stateless-решение для защиты роутов; refresh-токен продлевает сессию без повторного логина и без серверного стора сессий. |

## 3. Этапы разработки

1. **Проектирование.** Определены роли, экраны и сквозной сценарий
   «создание → запуск → прохождение → результаты». Спроектированы модель данных
   (7 сущностей, см. [DATABASE.md](DATABASE.md)) и протокол WebSocket-событий
   (см. [ARCHITECTURE.md](ARCHITECTURE.md)).
2. **Модель БД.** Prisma-схема (SQLite), миграции, seed с демо-организатором и
   готовым квизом для мгновенной проверки.
3. **Backend — REST.** Аутентификация (bcrypt + JWT, middleware ролей и владения),
   CRUD квизов и вопросов с валидацией (2–6 вариантов, корректность правильных
   ответов, обязательное изображение для image-вопроса, диапазон времени),
   загрузка изображений (multer, локальный диск с абстракцией под облако),
   эндпоинты истории для кабинетов.
4. **Backend — реальное время.** Менеджер живых сессий: комнаты по коду,
   серверный таймер как источник истины, приём/валидация ответов, подсчёт баллов
   (базовые + бонус за скорость, «всё-или-ничего» для множественного выбора),
   промежуточный и финальный лидерборд, сохранение результатов, безопасное
   переподключение участника (секрет `rejoinToken`, дедуп по пользователю).
5. **Frontend.** Каркас (роутинг, auth-контекст с авто-refresh токена и корректным
   выходом при истечении сессии), страницы входа/регистрации, кабинеты обеих
   ролей, редактор квизов с формой вопросов и загрузкой картинок, экран ведущего
   (управление сессией), экран игрока (ответы, таймер, показ правильного ответа,
   live-лидерборд), синхронизация клиентского таймера с сервером. Интерфейс
   оформлен в фирменном стиле VK.
6. **Тестирование.** Три автоматических набора проверок в `client/`:
   - `test:api` — REST: роли, guard-ы доступа, CRUD, валидация, загрузка
     изображений, история кабинетов;
   - `test:e2e` — realtime: организатор + 2 участника через WebSocket, серверный
     таймер, подсчёт баллов, финальный лидерборд, сохранение результатов;
   - `test:security` — защита личности участника (предотвращение подмены,
     переподключение по `rejoinToken`, дедуп авторизованного участника).

## 4. Соответствие требованиям задания

| Требование | Реализация |
|------------|------------|
| Регистрация/авторизация участников и организаторов | JWT-аутентификация, роли, гостевой вход участников |
| Создание и настройка квизов (категории, время, правила) | Редактор квизов + настройки (категория, время на вопрос, смена ответа, бонус за скорость) |
| Вопросы разных типов (текст/изображение, одиночный/множественный) | Поддержаны все четыре комбинации, 2–6 вариантов, загрузка изображений |
| Запуск, подключение по коду, real-time показ, ответ только во время демонстрации | Socket.IO-комнаты, серверный таймер, отклонение поздних ответов |
| Подсчёт баллов и определение победителей | Баллы с бонусом за скорость, финальный лидерборд с победителем |
| Личный кабинет с историей | Кабинеты организатора и участника с историей и результатами |
| Работа через WebSocket | Socket.IO на клиенте и сервере |
